## Objective

When this spec is done, the repo contains a complete, buildable, deployable TypeScript project skeleton: exact-pinned `package.json` + committed lockfile, a fully specified ESM/NodeNext `tsconfig.json`, `railway.json`, `.env.example`, `.gitignore`, `.npmrc`, a `src/config.ts` that validates every environment variable at boot with zod and fails fast, a minimal placeholder `src/server.ts` (Fastify + `GET /health` only) so `npm run build && npm start` produces a healthy process, a `src/logger.ts` stub defining the import boundary for Spec 08, and a working zero-dependency test harness. This spec resolves gaps G1 (module system), G2 (.env loading), and G6 (test runner) from findings/10.

## Requirements

### R1 ‚Äî Module system: ESM (`"type": "module"`) with NodeNext (RESOLVES G1)

The project is **ESM**: `"type": "module"` in `package.json`, `"module": "NodeNext"` + `"moduleResolution": "NodeNext"` in `tsconfig.json`, and **every relative import in `src/` carries an explicit `.js` extension** (e.g. `import { loadConfig } from './config.js';` ‚Äî the `.js` refers to the emitted file; NodeNext enforces this at compile time).

Justification against the findings (this is a committed decision, not a preference):

1. The MCP SDK's canonical deep imports (`@modelcontextprotocol/sdk/server/mcp.js`, `.../server/streamableHttp.js`, `.../client/index.js`, `.../client/streamableHttp.js`) are **spike-verified working under Node 22 ESM with TS `moduleResolution: nodenext`** [findings/05 C12]. That is the exact combination this spec pins ‚Äî no other tsconfig combination in this repo has findings backing.
2. The one known ESM hazard ‚Äî `import { mulaw } from 'alawmulaw'` crashes under Node ESM named imports [findings/06 C8, gotcha 1] ‚Äî is **removed entirely** because findings/06 recommends vendoring the ~30-line Œº-law tables instead of depending on `alawmulaw` (the DSP spec MUST vendor; `alawmulaw` is NOT in `package.json`). With that dependency gone, no package in the pinned set has an ESM interop problem: `fastify@5.10.0`, `@fastify/websocket@11.3.0`, `@fastify/formbody@8.0.2`, `@ai-sdk/gateway@4.0.23`, `zod`, and the MCP SDK all ship proper `exports` maps; `ws@8.21.1` and `twilio@6.0.2` are CJS consumed via default import, handled by `esModuleInterop` (the exact import forms `import WebSocket from 'ws'` and `import twilio from 'twilio'` appear in findings 01/08 verified snippets).
3. ESM permits top-level `await` in `src/server.ts` (`await app.register(...)`, `await app.listen(...)`), which is the shape of every verified boot snippet in findings/08.
4. `tsx` (dev runner) and `tsc` emit/execute this combination natively; Railpack's `npm run build` ‚Üí `tsc` ‚Üí `node dist/server.js` flow is unchanged [findings/07 claim 5].

Consequences for every later spec: all relative imports use `.js` extensions; never use `require()`; never import `alawmulaw` (vendor the tables per findings/06).

### R4 ‚Äî `.env` loading via Node's built-in `--env-file` (RESOLVES G2)

**Decision: Node 22's native `--env-file` flag. `dotenv` is NOT installed.** Rationale: zero dependencies, no import-order footgun (env is populated before any module executes, so `config.ts` can read `process.env` at load time), and it composes with the dev runner: `tsx watch --env-file=.env src/server.ts` passes the flag through to Node [findings/10 G2].

Rules:

1. `npm run dev` requires a local `.env` file (created by copying `.env.example`); Node errors clearly if the file is missing ‚Äî acceptable, since a dev run without env vars would only fail later and worse (see R5).
2. Production (`npm start` on Railway) uses **no** env-file flag: Railway injects variables directly into the process environment [findings/07 claims 8, 14]. Sealed variables are unavailable via `railway run`, so local dev MUST use `.env`, never `railway run` [findings/07 claim 14].
3. `npm test` deliberately does NOT load `.env` ‚Äî tests must pass with an empty environment (they call `loadConfig` with explicit fixture objects, R5).
4. `.env` is gitignored (R9); `.env.example` (R8) is the committed template.

### R5 ‚Äî `src/config.ts`: zod-validated, fail-fast boot configuration

`config.ts` exports a pure function plus a type ‚Äî no side effects at import time (so tests can import it with any fixture env, and no jsdom-adjacent surprise ever touches it):

```ts
import { z } from 'zod';

const EnvSchema = z.object({
  AI_GATEWAY_API_KEY: z
    .string()
    .min(1, 'AI_GATEWAY_API_KEY is required (Vercel dashboard ‚Üí AI Gateway ‚Üí API Keys). ' +
      'Without it the SDK silently falls back to Vercel OIDC, which fails late and obscurely off-Vercel.'),
  TWILIO_AUTH_TOKEN: z.string().min(1, 'TWILIO_AUTH_TOKEN is required (Twilio Console dashboard).'),
  PORT: z.coerce.number().int().positive().default(3000),
  RAILWAY_PUBLIC_DOMAIN: z.string().min(1).optional(),
  PUBLIC_HOST: z.string().min(1).optional(),
  MODEL_ID: z.string().min(1).default('openai/gpt-realtime-2.1'),
  AUDIO_MODE: z.enum(['pcmu', 'transcode']).default('transcode'),
  VOICE: z.string().min(1).default('marin'),
});

export interface AppConfig {
  aiGatewayApiKey: string;
  twilioAuthToken: string;
  port: number;
  /** Bare hostname (no scheme) used for wss:// and https:// URL construction.
   *  PUBLIC_HOST (local/ngrok) wins over RAILWAY_PUBLIC_DOMAIN (injected by Railway). */
  publicHost: string;
  modelId: string;
  audioMode: 'pcmu' | 'transcode';
  voice: string;
}

export function loadConfig(env: NodeJS.ProcessEnv = process.env): AppConfig {
  const parsed = EnvSchema.safeParse(env);
  if (!parsed.success) {
    const lines = parsed.error.issues.map(i => `  - ${i.path.join('.') || '(env)'}: ${i.message}`);
    throw new Error(`Invalid environment configuration:\n${lines.join('\n')}`);
  }
  const e = parsed.data;
  const publicHost = e.PUBLIC_HOST ?? e.RAILWAY_PUBLIC_DOMAIN;
  if (!publicHost) {
    throw new Error(
      'Invalid environment configuration:\n  - set PUBLIC_HOST (local dev, e.g. your ngrok host) ' +
      'or run on Railway where RAILWAY_PUBLIC_DOMAIN is injected after Generate Domain.');
  }
  return {
    aiGatewayApiKey: e.AI_GATEWAY_API_KEY,
    twilioAuthToken: e.TWILIO_AUTH_TOKEN,
    port: e.PORT,
    publicHost,
    modelId: e.MODEL_ID,
    audioMode: e.AUDIO_MODE,
    voice: e.VOICE,
  };
}
```

Requirements on this module:

1. **Fail fast at boot**: `src/server.ts` calls `loadConfig()` as its first action inside a try/catch that prints the error message to stderr and `process.exit(1)`. This is a correctness feature, not a convenience: a missing `AI_GATEWAY_API_KEY` otherwise surfaces mid-call as a confusing `GatewayAuthenticationError` about OIDC/`vc env pull` [findings/01 claim 14, gotcha 5; findings/10 G2]. A boot crash on Railway also fails the deploy healthcheck, so a mis-configured push never replaces the live deployment [findings/07 claim 11].
2. `publicHost` is a **bare hostname, no scheme** ‚Äî `RAILWAY_PUBLIC_DOMAIN` is injected in that form [findings/07 claim 7]. Later specs build `https://${publicHost}/twiml` and `wss://${publicHost}/twilio-media` from it and MUST NOT derive URLs from request headers [findings/08 gotcha 6].
3. `PORT` must never be set as a Railway service variable (Railway provides it); the default 3000 is for local dev [findings/07 claim 8, gotcha 6].
4. `AUDIO_MODE` defaults to `transcode` (Path B baseline per BRD ¬ß12); flipping to `pcmu` after the M1 spike (S1) is a Railway variable change, not a code change.
5. `VOICE` defaults to `marin` per BRD ¬ß12, but `marin` is **unverified through the gateway** [findings/10 G3] ‚Äî the M1 spike (S8) confirms it via `session-updated.raw`, with `alloy` as the fallback value. The config layer treats it as an opaque string either way.
6. `MODEL_ID` default `openai/gpt-realtime-2.1`; fallback `openai/gpt-realtime-2` is a variable change (spike S7).

### R7 ‚Äî Test runner: `node:test` executed through tsx (RESOLVES G6)

**Decision: Node's built-in `node:test`, run as `tsx --test "src/**/*.test.ts"`. Vitest is NOT installed.**

Justification (G6's hard requirement is "must run in a node environment, never jsdom" [findings/10 G6; findings/01 gotcha 6 ‚Äî `getToken` throws wherever `globalThis.window` is defined]):

1. `node:test` runs in the plain Node environment **structurally** ‚Äî there is no `environment` config knob that a future edit could flip to jsdom. The G6 trap becomes unrepresentable rather than merely avoided.
2. Zero new dependencies and zero unpinned versions: `tsx` is already in devDependencies for `npm run dev`, and `tsx --test` compiles TS test files and delegates to Node's test runner.
3. The test surface this project needs is assertion-heavy and framework-light: the DSP suite ported from findings/06 (codec round-trip, chunk-boundary bit-identity, click detector, tone-projection THD+N ‚Äî all plain numeric asserts with `node:assert/strict`) and config/unit tests. Nothing needs vitest's mocking, snapshots, or watch UI.

Conventions for all specs: test files live next to sources as `src/<name>.test.ts`, import from `node:test` (`describe`, `it`) and `node:assert/strict`, and use explicit `.js` extensions on relative imports like all other files (R1).

Deliverable `src/config.test.ts` ‚Äî proves the harness and the fail-fast contract:

```ts
import { describe, it } from 'node:test';
import assert from 'node:assert/strict';
import { loadConfig } from './config.js';

const BASE = {
  AI_GATEWAY_API_KEY: 'vck_test',
  TWILIO_AUTH_TOKEN: 'tok_test',
  PUBLIC_HOST: 'example.ngrok.app',
};

describe('loadConfig', () => {
  it('applies defaults', () => {
    const c = loadConfig({ ...BASE });
    assert.equal(c.port, 3000);
    assert.equal(c.modelId, 'openai/gpt-realtime-2.1');
    assert.equal(c.audioMode, 'transcode');
    assert.equal(c.voice, 'marin');
    assert.equal(c.publicHost, 'example.ngrok.app');
  });
  it('throws with a clear message when AI_GATEWAY_API_KEY is missing', () => {
    const { AI_GATEWAY_API_KEY: _omit, ...env } = BASE;
    assert.throws(() => loadConfig(env), /AI_GATEWAY_API_KEY/);
  });
  it('prefers PUBLIC_HOST over RAILWAY_PUBLIC_DOMAIN', () => {
    const c = loadConfig({ ...BASE, RAILWAY_PUBLIC_DOMAIN: 'x.up.railway.app' });
    assert.equal(c.publicHost, 'example.ngrok.app');
  });
  it('throws when neither PUBLIC_HOST nor RAILWAY_PUBLIC_DOMAIN is set', () => {
    const { PUBLIC_HOST: _omit, ...env } = BASE;
    assert.throws(() => loadConfig(env), /PUBLIC_HOST|RAILWAY_PUBLIC_DOMAIN/);
  });
  it('rejects an invalid AUDIO_MODE', () => {
    assert.throws(() => loadConfig({ ...BASE, AUDIO_MODE: 'wav' }));
  });
});
```

### R11 ‚Äî `src/server.ts` minimal placeholder

This spec delivers only enough server to make the pipeline provable end-to-end (build ‚Üí start ‚Üí healthcheck). Later specs replace/extend it; they MUST preserve these invariants:

```ts
import Fastify from 'fastify';
import { loadConfig, type AppConfig } from './config.js';
import { logEvent } from './logger.js';

let config: AppConfig;
try {
  config = loadConfig();
} catch (err) {
  console.error(err instanceof Error ? err.message : String(err));
  process.exit(1);
}

const app = Fastify({ logger: false, trustProxy: true }); // structured logging is Spec 08's logEvent, not pino transport

app.get('/health', async () => ({ ok: true }));
```

### R12 ‚Äî `src/logger.ts` stub: the import boundary only (full spec is Spec 08)

This spec fixes the module's public surface so every other spec can import it today; Spec 08 owns the real implementation (event vocabulary, latency timestamps, rate discipline). The stub must already satisfy the Railway Log Explorer line contract so nothing ever needs re-plumbing [findings/07 claim 12]:

```ts
export type LogLevel = 'debug' | 'info' | 'warn' | 'error';

export interface LogFields {
  level: LogLevel;
  message: string;          // required by Railway's structured-log contract
  event: string;            // machine-readable event name (e.g. 'boot', 'stream-start')
  callSid?: string;         // ALWAYS top-level when present ‚Äî @callSid: filtering [findings/07 claim 12]
  streamSid?: string;
  [key: string]: unknown;   // additional flat attributes; keep values scalar, never nest
}

/** One minified JSON object per line to stdout. Never call per media frame (500 lines/s/replica cap). */
export function logEvent(fields: LogFields): void {
  process.stdout.write(JSON.stringify(fields) + '\n');
}
```

Boundary rules binding on all specs: import ONLY `logEvent` (and the types) from `./logger.js` until Spec 08 lands ‚Äî Spec 08 additionally exports `log(level, message, fields)`, `ms`, and `now`, and modules may then use either surface; never `console.log` structured data; never log per-frame; `callSid`/`streamSid` stay top-level flat keys. Spec 08 extends this file but must keep `logEvent` exported with this exact signature.
